# VoiceScape 产品设计文档 V1.0

## 一、产品定位

VoiceScape 是一款**纯语音驱动的对话式场景绘图工具**。

用户通过自然语言描述场景，系统将描述解析为可独立编辑的对象集合，并渲染在画布上。不同于传统 AI 生图工具输出不可编辑的整张图片，VoiceScape 生成的是由多个独立对象及其关系构成的**可编辑场景**。

---

## 二、设计原则

### 语音优先
所有创作行为均通过语音完成。鼠标/键盘仅用于：点击录音按钮触发录音、点击对象面板选中对象、点击导出按钮。

### 对象可编辑
生成结果不是整张图片，而是 `Object + Relation + Scene` 组成的场景，每个对象独立可修改。

### 自然语言优先
用户无需学习命令语法，系统理解自然语言意图。

### 低认知负担
复杂信息（对象 ID、坐标、关系表）由系统管理，用户只描述意图。

## 三、整体 UI 布局

```
┌─────────────────────────────────────────────────────┐
│                    顶部标题栏                         │
├──────────────┬──────────────────────────────────────┤
│              │                                      │
│   对象面板    │              主画布                   │
│  (Object     │            (Canvas)                  │
│   Panel)     │                                      │
│              │                                      │
│  - 对象列表   │   [AI对象图片] [AI对象图片]            │
│  - 关系列表   │                                      │
│  - 场景摘要   │         [AI背景图]                    │
│              │                                      │
├──────────────┴──────────────────────────────────────┤
│  文字反馈区（系统消息、追问、确认提示）                  │
├─────────────────────────────────────────────────────┤
│  [🎤 点击说话]  语音输入状态提示                        │
└─────────────────────────────────────────────────────┘
```

### 3.1 对象面板（左侧，约 25% 宽度）

展示三个区块：
- **当前对象**：列出所有场景对象，每项显示名称 + 缩略图，可点击选中
- **当前关系**：文字列表，如「我骑着马」「朋友站在房子旁边」
- **场景摘要**：一句话描述当前背景，如「魔法森林，黄昏」

点击某个对象后：
- 该对象在面板中高亮（蓝色边框）
- 画布上对应对象高亮（发光/描边效果）
- 文字反馈区显示「已选中：马1，接下来的语音命令将针对它」
- 再次点击取消选中

### 3.2 主画布（中间，约 75% 宽度）

- 默认白色背景
- 背景确认后，AI 生成背景图铺满画布底层
- 对象图片（去背景的 PNG）叠加在背景上方
- 对象可通过语音命令移动位置
- 选中对象显示高亮边框

### 3.3 文字反馈区（底部，固定高度约 80px）

系统所有回复均在此以文字形式显示，包括：
- 操作确认：「已创建：马（马1）」
- 追问：「你希望背景是什么环境？」
- 风险确认：「将删除所有树木，确认吗？请说"确认"或"取消"」
- 错误提示：「没有找到你描述的对象，能更具体说明吗？」

### 3.4 语音输入区（最底部）

- 核心元素：一个大录音按钮「🎤 点击说话」
- 点击后变为「🔴 录音中... 再次点击停止」
- 录音时按钮有动态波形动画
- 发送后显示「⏳ 处理中...」直到系统响应完成

## 四、核心交互流程

### 4.1 首次场景创建

```
用户点击录音按钮
  ↓
用户说「我梦见自己骑马去朋友家」
  ↓
用户再次点击停止录音
  ↓
STT 转为文字
  ↓
LLM 解析场景：提取对象（我/马/朋友/房屋）、关系（我骑着马）
  ↓
AI 生成各对象的去背景图片（扁平插画风格）
  ↓
对象渲染到画布，自动布局
  ↓
文字反馈区显示「已创建 4 个对象：我、马1、朋友、房屋」
  ↓
系统追问：「你希望场景的背景是什么环境？」
  ↓
用户录音回答「傍晚的草原」
  ↓
AI 生成背景图，铺到画布底层
  ↓
反馈区：「背景已设置：傍晚的草原」
```

### 4.2 对象编辑流程

```
用户录音「把马移到房子前面」
  ↓
LLM 解析：操作=移动，对象=马1，位置=房屋前方
  ↓
画布更新：马1 图片移动到房屋图片前方（z 轴较高层）
  ↓
反馈区：「已移动：马1 → 房屋前方」
```

```
用户录音「把马改成白色」
  ↓
LLM 解析：操作=修改颜色，对象=马1，颜色=白色
  ↓
AI 重新生成白色马的图片
  ↓
画布更新：替换马1 图片
  ↓
反馈区：「已更新：马1 → 白色」
```

### 4.3 精准编辑（对象面板选中 + 语音）

```
用户在对象面板点击「马1」
  ↓
马1 在画布上高亮（发光边框）
  ↓
反馈区：「已选中：马1，接下来的语音命令将针对它」
  ↓
用户录音「让它变大一些」
  ↓
LLM 知道当前选中马1，解析：操作=缩放，对象=马1，大小=放大
  ↓
画布更新：马1 图片尺寸增大 ~1.5x
  ↓
反馈区：「已缩放：马1 → 放大」
```

### 4.4 指代消解

有歧义时系统追问：

```
用户：「把树移到右边」（场景中有树1、树2）
  ↓
反馈区：「场景中有 树1 和 树2，你说的是哪一棵？」
  ↓
用户：「左边那棵」
  ↓
LLM 结合位置信息定位到 树1
  ↓
执行移动
```

无歧义时直接执行（L1 低风险操作）。

### 4.5 风险等级确认

| 等级 | 触发条件 | 系统行为 |
|------|---------|---------|
| L1 | 单对象低影响操作（移动、修改颜色） | 直接执行，反馈结果 |
| L2 | 批量操作或删除（删除所有树木） | 先反馈操作内容，追问「确认吗？说"确认"或"取消"」 |
| L3 | 大范围场景重构（将场景改为未来城市） | 拆解步骤列出，逐条确认后执行 |

## 五、对象系统设计

### 5.1 对象视觉呈现

- 每个对象由 AI 生成一张**去背景的 PNG 图片**
- 风格：**扁平插画风（Flat Illustration）**，全场景统一
- 生成 Prompt 约束：`flat illustration style, no background, transparent background, consistent cartoon style`
- 对象尺寸：默认为画布高度的 20~30%，用户可通过语音调整

### 5.2 对象命名规则

| 场景 | 命名规则 | 示例 |
|------|---------|------|
| 首次出现，无特征 | 类型名 | 马 |
| 同类出现 ≥2 个，有特征 | 特征+类型名 | 白马、黑马 |
| 同类出现 ≥2 个，无特征 | 类型名+序号 | 马1、马2 |
| 创建时用户已描述特征 | 直接用特征命名 | 白色的马 → 白马 |

内部以唯一 ID 追踪（如 `horse_001`），面板展示用户友好名称。

### 5.3 关系在画布上的表达

关系通过**对象的相对位置与层叠**来体现：

| 关系类型 | 画布表现 |
|---------|---------|
| 骑着（人骑马） | 人物图片叠在马图片上方，略偏后 |
| 站在旁边 | 两对象紧靠放置 |
| 拿着 | 物品图片叠在人物手部区域 |
| 前面/后面 | 通过 z-index 和垂直位置区分（前=下方=z 高） |
| 左边/右边 | 水平位置调整 |

关系同时在对象面板的「当前关系」区块中以文字显示。

### 5.4 对象初始布局规则

- 新对象出现位置：画布中心区域，AI 根据描述判断大致位置
- 多对象时：自动分散布局，避免完全重叠
- 有明确关系时（如「我骑着马」）：两对象按关系规则叠放

## 六、语音输入设计

### 6.1 触发方式
- 点击录音按钮 → 开始录音
- 再次点击 → 停止录音并提交

### 6.2 录音状态视觉反馈

| 状态 | 按钮样式 | 提示文字 |
|------|---------|------|
| 待机 | 灰色麦克风图标 | 「点击说话」 |
| 录音中 | 红色脉冲动画 | 「录音中... 再次点击停止」 |
| 处理中 | 旋转加载动画 | 「处理中...」 |
| 完成 | 恢复灰色 | 「点击说话」 |

## 七、场景状态模型（Scene State）

系统内部维护一份 JSON 状态，驱动所有渲染：

```json
{
  "scene_summary": "傍晚的草原",
  "background_image_url": "https://...",
  "objects": [
    {
      "id": "horse_001",
      "display_name": "马1",
      "type": "horse",
      "image_url": "https://...",
      "position": { "x": 400, "y": 300 },
      "size": { "width": 200, "height": 150 },
      "z_index": 2,
      "attributes": { "color": "brown" }
    }
  ],
  "relations": [
    {
      "subject_id": "person_001",
      "relation": "骑着",
      "object_id": "horse_001"
    }
  ]
}
```

## 八、导出功能

MVP 仅支持「导出为图片」：
- 顶部标题栏提供「📥 导出图片」按钮
- 点击后将整个画布（包含背景 + 所有对象）导出为 PNG 文件
- 文件名格式：`voicescape_场景名_日期.png`
- 不支持会话保存/恢复（MVP 范围外）

## 九、MVP 功能范围

### 包含

| 功能 | 说明 |
|------|------|
| 语音输入 | 点击触发，STT 转文字 |
| 场景解析 | LLM 提取对象和关系 |
| AI 对象生图 | 扁平插画风，去背景 PNG |
| 背景追问+生成 | 首次建场景后追问背景 |
| 创建对象 | 支持 |
| 删除对象 | 支持（L2 风险确认） |
| 移动对象 | 支持位置描述 |
| 缩放对象 | 支持「放大/缩小」 |
| 修改颜色 | 重新生成对应颜色图片 |
| 关系编辑 | 支持位置关系和依附关系 |
| 对象面板 | 展示 + 点击选中 |
| 文字反馈 | 所有系统消息文字显示 |
| 指代消解 | 有歧义时追问 |
| 风险确认 | L1/L2/L3 三级 |
| 导出图片 | PNG 导出 |

### 不包含（MVP 范围外）

- 旋转对象
- 修改风格（如童话风 → 写实风）
- 表情修改
- 会话保存/恢复
- 故事时间线
- 世界状态回溯
- 3D 场景
- 语音播报（TTS）
